# Antigravity Resource Management Rules

[Strict Token Conservation: ON]

- **Respuetas Concisas:** No saludes, no pidas disculpas ni uses frases de cortesía innecesarias como "Espero que esto te sirva". Ve directo al grano.
- **Diferencial de Código:** Al editar archivos, NO reescribas el archivo completo. Proporciona solo el bloque de código modificado o usa el formato `// ... código existente` para omitir partes que no han cambiado.
- **Lectura Selectiva:** No escanees directorios completos a menos que sea estrictamente necesario para la lógica del problema. Prioriza archivos abiertos.
- **Pensamiento Compacto:** Si usas modelos con "Reasoning" (como Gemini 3.1 Pro o Claude), limita el proceso de pensamiento a la validación técnica, evitando resúmenes narrativos de lo que vas a hacer.
- **Formato de Salida:** Usa listas cortas y tablas solo si ahorran espacio. Evita explicaciones teóricas extensas a menos que se soliciten explícitamente.
